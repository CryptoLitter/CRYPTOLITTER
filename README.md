# CryptoLitter GitHub Pages Site

## Before publishing
Edit `config.js` and enter only verified official values:
- contractAddress
- buyUrl
- xUrl
- telegramUrl
- projectStatus

The buttons remain disabled while values are blank. Review the proposed token allocation in `index.html` and change it if your final plan differs.

## Publish on GitHub Pages
1. Sign in to GitHub.
2. Create a new **public** repository named `cryptolitter`.
3. Extract the ZIP and upload all files inside the `cryptolitter-site` folder to the repository root.
4. Commit the files.
5. Open **Settings → Pages**.
6. Under **Build and deployment**, choose **Deploy from a branch**.
7. Select branch **main** and folder **/(root)**, then save.
8. GitHub will provide a URL similar to `https://YOUR-USERNAME.github.io/cryptolitter/`.

## Safety checklist
- Verify source code on the official explorer.
- Publish exact allocation wallet addresses.
- Publish liquidity transaction and lock proof.
- Never claim Robinhood endorsement or guaranteed listing.
- Never promise profits.
